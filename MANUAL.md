## 1. 프로그램 개요

**미체결DB 배정** 프로그램은 SQL Server의 `TM_MEMBER` 테이블에서 미체결(홈쇼핑 채널, 미등록) DB를 조회하고, 홈쇼핑 TM 담당자에게 라운드로빈 방식으로 균등 배정한 뒤 DB를 일괄 업데이트하는 업무 자동화 도구입니다.

### 주요 기능 요약

| 버튼 | 기능 |
|------|------|
| DB 조회 | 월별 미체결 건수 집계 |
| 상세 조회 | 선택 월의 상세 DB 및 담당자 목록 조회 |
| DB 배정 | 라운드로빈 방식으로 담당자 자동 배정 |
| DB 입력 | DB UPDATE + 엑셀 파일 자동 생성 |

## 4. 기능 설명

### 4-1. DB 조회

**버튼**: `DB 조회` (파란색)

실행 SQL:
```sql
SELECT LEFT(Rec_Date,7) AS Rec_YM, COUNT(OrderNo) AS CNT
FROM TM_MEMBER
WHERE HS_Name LIKE '%쇼핑%' AND Mem_ID = ''
GROUP BY LEFT(Rec_Date,7)
ORDER BY LEFT(Rec_Date,7)
```

- 홈쇼핑 채널(`HS_Name LIKE '%쇼핑%'`)이며 미등록(`Mem_ID = ''`) 건을 월별로 집계합니다.
- 결과는 **월별 DB 수량** 테이블에 표시됩니다.
- 각 행 왼쪽의 체크박스로 상세조회할 월을 선택합니다.

---

### 4-2. 상세 조회

**버튼**: `상세 조회` (녹색)

**전제 조건**: 월별 DB 수량 테이블에서 1개 이상의 월을 체크해야 합니다.  
체크된 월이 없으면 **"[월]을 선택하세요"** 팝업이 표시됩니다.

실행 SQL (선택 월 변수 적용):
```sql
SELECT DISTINCT
    REPLACE(Mobile,'-','') AS Mobile,
    ID, Name, Rec_Date, MemoType,
    AssignCharge_ID, saname, PlaceofDuty,
    AssignDate, OrderNo, HS_Name
FROM dbo.TM_MEMBER
WHERE LEFT(Rec_Date,7) IN ('2021-10','2022-02', ...)
ORDER BY Rec_Date, ID
```

동시에 배정 담당자 조회:
```sql
SELECT SaBun, SaName FROM Staff
WHERE PlaceofDuty IN ('홈쇼핑 TM','미채결')
AND OutDate = '' AND BranchOffice IN ('TM1','TM2')
ORDER BY PlaceofDuty DESC, SaName
```

- **DB 내용** 테이블: 상세 조회 결과 표시 (신규배정일, 신규담당자 컬럼은 빈값)
- **배정** 테이블: 배정 가능한 담당자 목록 표시 (배정갯수는 0으로 초기화)

---

### 4-3. DB 배정

**버튼**: `DB 배정` (주황색)

**전제 조건**: 배정 테이블에서 1명 이상의 담당자에게 배정갯수 (1 이상)를 입력해야 합니다.

**배정 방식**: 라운드로빈 (Round-Robin)

예시) 가미숙 4건, 박미아 2건 배정:
```
순서 1: 가미숙
순서 2: 박미아
순서 3: 가미숙
순서 4: 박미아
순서 5: 가미숙
순서 6: 가미숙
```

- DB 내용 테이블의 **첫 번째 행부터** 순서대로 배정됩니다.
- 배정된 행: `신규배정일` = 오늘 날짜(YYYY-MM-DD), `신규담당자` = 담당자명
- 배정된 행의 배경색이 변경되어 시각적으로 구분됩니다.
- 배정 총 건수가 DB 내용 건수를 초과하면 경고 팝업이 표시됩니다.

> **주의**: DB 배정은 화면상 처리만 수행합니다. 실제 DB 반영은 **DB 입력** 버튼을 눌러야 합니다.

---

### 4-4. DB 입력

**버튼**: `DB 입력` (빨간색)

**전제 조건**: DB 배정이 완료된 상태여야 합니다.

실행 내용:
1. 입력 확인 팝업 표시 (배정 건수 확인)
2. SQL UPDATE 실행:
   ```sql
   UPDATE TM_MEMBER
   SET AssignCharge_ID = '사번', AssignDate = '배정일'
   WHERE ID = '회원ID'
   ```
3. 배정된 전체 건에 대해 일괄 UPDATE 후 COMMIT
4. **엑셀 파일 자동 생성** 및 저장

## 5. 배정 로직 상세

### 라운드로빈 알고리즘

```
입력: [(가미숙, 4), (박미아, 2)]

반복 1라운드: 가미숙(4→3), 박미아(2→1)  →  [가미숙, 박미아]
반복 2라운드: 가미숙(3→2), 박미아(1→0)  →  [가미숙, 박미아]
반복 3라운드: 가미숙(2→1), 박미아 완료  →  [가미숙]
반복 4라운드: 가미숙(1→0)               →  [가미숙]

최종 순서: 가미숙, 박미아, 가미숙, 박미아, 가미숙, 가미숙
```

- 배정 잔여 건수가 0인 담당자는 자동으로 건너뜁니다.
- DB 내용 테이블의 순서(Rec_Date, ID 기준 정렬)대로 배정됩니다.

## 8. 오류 대처

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| `Config_DB.db 를 다운로드할 수 없습니다` | Google Drive 접근 불가 또는 FILE_ID 오류 | `GDRIVE_URL` 상수의 파일 ID 확인, 인터넷 연결 확인 |
| `DBCON 테이블에 Name='HD_MSSQL' 레코드가 없습니다` | Config_DB.db 내 레코드 없음 | Config_DB.db에 해당 Name 행 INSERT 후 재업로드 |
| `[오류] 배정갯수를 입력해 주세요` | 배정 테이블 배정갯수 미입력 | 배정 담당자의 배정갯수 셀에 숫자 입력 |
| `[월]을 선택하세요` | 월별 DB 수량 체크박스 미선택 | 체크박스로 조회할 월 선택 |
| `배정 총 갯수가 DB 건수를 초과합니다` | 배정갯수 합계 > 조회 건수 | 배정 수량 조정 |
| ODBC 연결 오류 | ODBC 드라이버 미설치 또는 연결 정보 오류 | ODBC 드라이버 설치 확인, Config_DB.db 연결 정보 검증 |
| `DB 배정 먼저 실행하세요` | 배정 없이 DB 입력 시도 | DB 배정 버튼 먼저 실행 |

### Config_DB.db 수동 갱신

DB 연결 정보가 변경된 경우:
1. `./DB/Config_DB.db` 파일 삭제
2. 프로그램 재실행 → 자동으로 최신 파일 다운로드

## 주의사항

- `./DB/` 폴더 및 `Config_DB.db` 는 **절대 Git에 커밋하지 마세요** (DB 비밀번호 포함).
