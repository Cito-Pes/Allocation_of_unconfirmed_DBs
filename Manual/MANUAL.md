## 1. 프로그램 소개

**미체결DB 배정** 프로그램은 SQL Server에 저장된 홈쇼핑 미체결 고객 DB를 조회하여,  
홈쇼핑 TM 팀 담당자에게 **라운드로빈 방식**으로 균등하게 배정한 뒤  
결과를 DB에 일괄 저장하고 엑셀 파일로 출력하는 업무 자동화 도구입니다.

### 주요 특징

- **중앙 집중 설정 관리** : DB 접속 정보를 Google Drive의 `Config_DB.db`로 관리하여 소스코드에 비밀번호를 노출하지 않습니다.
- **라운드로빈 배정** : 여러 담당자에게 입력한 건수만큼 순서를 돌아가며 공평하게 배정합니다.
- **백그라운드 처리** : DB 조회 중에도 화면이 멈추지 않습니다.
- **엑셀 자동 저장** : DB 입력 완료 시 전체 시트와 담당자별 시트로 구성된 엑셀 파일을 자동 생성합니다.

## 2. 설치 및 초기 설정

### 2-1. 필요 소프트웨어

| 항목 | 버전 | 비고 |
|------|------|------|
| Python | 3.9 이상 (권장 3.13) | python.org |
| SQL Server ODBC 드라이버 | SQL Server Native Client 10.0 또는 ODBC Driver 17 | Microsoft 공식 배포 |

### 2-2. Python 패키지 설치

명령 프롬프트(cmd)를 열고 프로젝트 폴더에서 아래 명령을 실행합니다.

```bash
pip install -r requirements.txt
```

설치되는 패키지:

| 패키지 | 용도 |
|--------|------|
| `PySide6` | UI 화면 구성 |
| `pyodbc` | SQL Server 접속 |
| `requests` | Google Drive 파일 다운로드 |
| `openpyxl` | 엑셀 파일 생성 |

### 2-3. Google Drive Config_DB.db 설정

프로그램은 처음 실행할 때 `./DB/Config_DB.db` 파일이 있는지 확인합니다.  
파일이 없으면 Google Drive에서 **자동으로 다운로드**합니다.

![Config DB 다운로드 흐름](images/manual/config_flow.svg)

**`main.py` 상단에서 수정해야 할 부분:**

```python
# 실제 Google Drive 공유 링크로 교체하세요
GDRIVE_URL = "https://drive.google.com/file/d/{실제_FILE_ID}/view?usp=drive_link"
```

Config_DB.db 내용이 변경된 경우 `./DB/Config_DB.db` 파일을 삭제하고 프로그램을 재실행하면 최신 파일을 자동으로 내려받습니다.

### 2-4. 프로그램 실행

```bash
python main.py
```

화면은 크게 **4개 영역**으로 나뉩니다.

| 번호 | 영역 | 설명 |
|------|------|------|
| ① | **월별 DB 수량** | DB 조회 결과. 체크박스로 상세 조회할 월을 선택합니다. |
| ② | **DB 내용** | 선택한 월의 고객 상세 정보. 배정 후 `신규배정일`, `신규담당자` 컬럼이 채워집니다. |
| ③ | **배정** | 배정 가능한 담당자 목록. `배정갯수` 셀에 직접 숫자를 입력합니다. |
| ④ | **처리상황** | 모든 작업의 실행 일시와 결과가 실시간으로 기록됩니다. |

| 컬럼 | 설명 |
|------|------|
| 체크박스 | 클릭하여 상세 조회할 월을 선택합니다. 복수 선택 가능합니다. |
| 월 | 년-월 형식 (예: 2021-10) |
| 수량 | 해당 월의 미체결 건수 |

### DB 내용 테이블 ②

| 컬럼 | 원본 DB 컬럼 | 설명 |
|------|-------------|------|
| 휴대전화 | Mobile | 하이픈(`-`) 제거 후 표시 |
| ID | ID | 회원 고유 ID (UPDATE 시 WHERE 조건으로 사용) |
| 성명 | Name | 회원 이름 |
| 등록일 | Rec_Date | DB 등록 날짜 |
| 상태 | MemoType | 현재 상태 (취소/부재 등) |
| 사번 | AssignCharge_ID | 현재 배정된 담당자 사번 |
| 담당자명 | saname | 현재 배정된 담당자명 |
| 권 | PlaceofDuty | 소속 채널 |
| 배정일 | AssignDate | 현재 배정일 |
| 주문번호 | OrderNo | 주문 번호 |
| 채널 | HS_Name | 홈쇼핑 채널명 |
| **신규배정일** | — | DB 배정 버튼 클릭 후 자동 입력 (오늘 날짜) |
| **신규담당자** | — | DB 배정 버튼 클릭 후 자동 입력 (배정된 담당자명) |

배정이 완료된 행은 **녹색 배경**으로 강조 표시됩니다.

### 배정 테이블 ③

| 컬럼 | 설명 |
|------|------|
| 사번 | DB UPDATE 시 `AssignCharge_ID` 값으로 사용 |
| 담당자명 | 담당자 이름 |
| 배정갯수 | **직접 입력** — 셀을 더블클릭하고 숫자 입력 (0이면 배정 제외) |

## 4. 버튼 안내

![버튼 안내](images/manual/buttons.svg)

| 버튼 | 색상 | 기능 | DB 변경 여부 |

**DB 입력** 버튼은 실제 SQL Server 데이터를 변경합니다. 배정 내용을 충분히 확인한 후 실행하세요.

## 5. 단계별 사용 방법

![전체 작업 순서](images/manual/workflow.svg)

반드시 **① → ② → ③ → ④ 순서**로 진행해야 합니다.

### STEP 1 – DB 조회

**목적** : 홈쇼핑 미체결 고객 DB의 월별 건수를 확인합니다.

**실행 방법**

1. 프로그램을 실행합니다.
2. 처리상황 로그에서 **"✓ 설정 로드 완료"** 메시지를 확인합니다.
3. 화면 하단의 **`DB 조회`** 버튼(파란색)을 클릭합니다.
4. 잠시 후 ① 월별 DB 수량 테이블에 결과가 채워집니다.

**실행되는 쿼리**

```sql
SELECT LEFT(Rec_Date,7) AS Rec_YM, COUNT(OrderNo) AS CNT
FROM TM_MEMBER
WHERE HS_Name LIKE '%쇼핑%' AND Mem_ID = ''
GROUP BY LEFT(Rec_Date,7)
ORDER BY LEFT(Rec_Date,7)
```

- `HS_Name LIKE '%쇼핑%'` : 홈쇼핑 채널 필터
- `Mem_ID = ''` : 미등록 고객 필터 (회원 전환 전)

**결과 확인**

```
2026-04-08 16:10:25    월별 DB 수량 조회 중...
2026-04-08 16:10:26    월별 DB 수량 조회 완료  (24건)
```

---

### STEP 2 – 상세 조회

**목적** : 배정 대상 고객의 상세 정보와 배정 가능한 담당자 목록을 불러옵니다.

**실행 방법**

1. ① 월별 DB 수량 테이블에서 상세 조회할 **월의 체크박스를 클릭**합니다.
   - 여러 개 선택 가능합니다.
2. **`상세 조회`** 버튼(녹색)을 클릭합니다.

체크된 월이 없으면 **"[월]을 선택하세요"** 팝업이 표시됩니다.

**실행되는 쿼리 (DB 내용)**

```sql
SELECT DISTINCT
    REPLACE(Mobile,'-','') AS Mobile,
    ID, Name, Rec_Date, MemoType,
    AssignCharge_ID, saname, PlaceofDuty,
    AssignDate, OrderNo, HS_Name
FROM dbo.TM_MEMBER
WHERE LEFT(Rec_Date,7) IN ('2021-10', '2022-02', ...)
ORDER BY Rec_Date, ID
```

**실행되는 쿼리 (배정 담당자)**

```sql
SELECT SaBun, SaName FROM Staff
WHERE PlaceofDuty IN ('홈쇼핑 TM', '미채결')
AND OutDate = ''
AND BranchOffice IN ('TM1', 'TM2')
ORDER BY PlaceofDuty DESC, SaName
```

**결과 확인**

```
2026-04-08 16:11:26    선택월 DB 상세조회 완료  (1509건)
2026-04-08 16:11:26    배정담당자 조회 완료  (6명)
```

---

### STEP 3 – DB 배정

**목적** : 담당자별 배정 건수를 입력하고, 라운드로빈 방식으로 자동 배정합니다.  
이 단계는 **화면에서만 처리**되며 실제 DB는 변경되지 않습니다.

**실행 방법**

1. ③ 배정 테이블에서 배정할 담당자의 `배정갯수` 셀을 **더블클릭**합니다.
2. 배정할 건수를 숫자로 입력하고 **Enter**를 누릅니다.
3. 배정할 모든 담당자에 대해 1~2번을 반복합니다.
4. **`DB 배정`** 버튼(주황색)을 클릭합니다.

배정갯수 합계가 DB 내용 건수를 초과하면 경고 팝업이 표시됩니다.

**배정 후 확인**

② DB 내용 테이블의 상단 행부터 순서대로 `신규배정일`과 `신규담당자`가 자동으로 입력되며, 배정된 행은 **녹색 배경**으로 강조됩니다.

```
2026-04-08 16:14:30    DB 배정 완료  (배정: 6건, 배정일: 2026-04-08)
```

---

### STEP 4 – DB 입력

**목적** : 배정된 정보를 실제 SQL Server DB에 저장하고, 엑셀 파일을 생성합니다.

**실행 방법**

1. STEP 3의 배정 결과를 충분히 검토합니다.
2. **`DB 입력`** 버튼(빨간색)을 클릭합니다.
3. 확인 팝업에서 배정 건수를 최종 확인하고 **"예"**를 클릭합니다.

**실행되는 쿼리**

```sql
UPDATE TM_MEMBER
SET AssignCharge_ID = '사번', AssignDate = '배정일'
WHERE ID = '회원ID'
```

각 배정 건에 대해 위 UPDATE를 실행한 후 일괄 COMMIT합니다.

**완료 후**

```
2026-04-08 16:15:11    ✓ DB 입력 완료  (6건)
2026-04-08 16:15:12    ✓ 엑셀 저장 완료: C:\Users\홍길동\Downloads\미체결_DB배정_2026-04-08.xlsx
```

## 6. 배정 방식 상세 설명

![라운드로빈 배정 방식](images/manual/roundrobin.svg)

### 라운드로빈(Round-Robin) 알고리즘

동일한 우선순위의 담당자들에게 순서를 돌아가며 배정하는 방식입니다.

**예시** : 가미숙 4건, 박미아 2건 배정 시

| 순번 | 배정 담당자 | 가미숙 잔여 | 박미아 잔여 |
|------|------------|-----------|-----------|
| 1번째 | 가미숙 | 3 | 2 |
| 2번째 | 박미아 | 3 | 1 |
| 3번째 | 가미숙 | 2 | 1 |
| 4번째 | 박미아 | 2 | 0 |
| 5번째 | 가미숙 | 1 | 완료 |
| 6번째 | 가미숙 | 0 | 완료 |

### 배정 규칙

- DB 내용 테이블의 **첫 번째 행부터** 순서대로 배정됩니다.
- DB 내용은 `Rec_Date`, `ID` 기준으로 정렬되어 있습니다.
- 배정갯수가 소진된 담당자는 자동으로 건너뜁니다.
- 배정 총 건수 이후의 행은 배정되지 않습니다 (`신규배정일`, `신규담당자` 공백).

## 7. 엑셀 파일 출력 안내

![엑셀 파일 구조](images/manual/excel_output.svg)

### 파일 정보

| 항목 | 내용 |
|------|------|
| 파일명 | `미체결_DB배정_{작업일}.xlsx` (예: `미체결_DB배정_2026-04-08.xlsx`) |
| 저장 위치 | `C:\Users\{사용자명}\Downloads\` |

### 시트 구성

| 시트명 | 포함 내용 |
|--------|----------|
| **전체** | 이번 작업에서 신규 배정된 전체 고객 데이터 |
| **{담당자명}** | 해당 담당자에게 배정된 고객만 필터링 (담당자 수만큼 생성) |

예를 들어 가미숙, 박미아에게 배정했다면 `전체`, `가미숙`, `박미아` 시트 3개가 생성됩니다.

### 파일 형식

- 헤더 : 파란색 배경, 흰색 볼드 글자, 맑은 고딕 9pt
- 데이터 : 홀수/짝수 행 교대 배경색
- 1행 고정 : 스크롤 시 헤더가 항상 보입니다
- 자동 필터 : 각 컬럼에 필터 버튼이 활성화됩니다
- 열 너비 : 내용에 맞게 자동 조정 (최대 30 문자)

## 9. 오류 메시지 및 해결 방법

### 시작 시 오류

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| Config_DB.db 를 다운로드할 수 없습니다 | Google Drive 접근 불가 또는 파일 ID 오류 | `main.py`의 `GDRIVE_URL` 확인, 인터넷 연결 확인 |
| Google Drive 파일 ID를 추출할 수 없습니다 | URL 형식 오류 | `GDRIVE_URL`이 올바른 Drive 공유 링크인지 확인 |
| DBCON 테이블에 Name='HD_MSSQL' 레코드가 없습니다 | Config_DB.db에 레코드 없음 | DB에 행 추가 후 Google Drive 재업로드, 로컬 파일 삭제 후 재실행 |
| ODBC 연결 오류 | 드라이버 미설치 또는 연결 정보 오류 | ODBC 드라이버 설치 확인, Config_DB.db의 접속 정보 검증 |

### 조회/배정/입력 시 오류

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| [월]을 선택하세요 | 체크박스 미선택 | 월별 DB 수량 테이블에서 월을 체크 후 재실행 |
| 배정갯수를 입력해 주세요 | 모든 배정갯수가 0 | 배정 테이블에서 1명 이상 숫자 입력 |
| 배정 총 갯수가 DB 건수를 초과합니다 | 배정 합계 > 조회 건수 | 배정 수량을 DB 건수 이하로 조정 |
| 배정된 데이터가 없습니다 | DB 배정 미실행 상태 | DB 배정 버튼 먼저 실행 |

### Config_DB.db 수동 갱신 방법

1. Windows 탐색기에서 `프로젝트폴더\DB\` 폴더를 엽니다.
2. `Config_DB.db` 파일을 삭제합니다.
3. 프로그램을 재실행하면 최신 파일이 자동으로 다운로드됩니다.

## 10. 자주 묻는 질문 FAQ

**Q. DB 배정 후 배정 내용이 틀렸습니다. 어떻게 하나요?**

DB 입력 버튼을 누르기 전이라면 상세 조회 버튼을 다시 클릭합니다. DB 내용 테이블이 초기화되므로 배정 테이블의 갯수를 다시 입력하고 DB 배정 버튼을 재실행하세요.

---

**Q. 배정갯수 합계가 DB 조회 건수보다 적어도 되나요?**

가능합니다. 배정되지 않은 행은 `신규배정일`, `신규담당자` 컬럼이 공백으로 남습니다. DB 입력 시에는 배정된 행만 UPDATE됩니다.

---

**Q. 엑셀 파일이 저장되지 않습니다.**

Downloads 폴더에 동일한 파일명이 Excel에서 열려 있는 경우 저장이 실패할 수 있습니다. Excel에서 해당 파일을 닫고 DB 입력을 다시 실행하세요.

---

**Q. 프로그램 실행 후 "설정 로드 완료" 메시지가 나오지 않습니다.**

처리상황 로그에서 빨간색 오류 메시지를 확인하세요. Google Drive 접근 문제라면 인터넷 연결을 확인하고, Config_DB.db의 FILE_ID가 올바른지 점검하세요.

---

**Q. SQL Server 2008 R2에서 어떤 ODBC 드라이버를 써야 하나요?**

**SQL Server Native Client 10.0** 사용을 권장합니다. `Config_DB.db`의 `DB_Type` 컬럼값을 드라이버명과 정확히 일치시켜야 합니다.

---

**Q. 동일한 날짜에 여러 번 DB 입력을 해도 되나요?**

파일명이 같으므로 Downloads 폴더의 기존 엑셀 파일이 덮어씌워집니다. 필요하면 기존 파일을 먼저 다른 이름으로 저장해 두세요.
