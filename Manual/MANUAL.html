1 프로그램 소개

미체결DB 배정 프로그램은 SQL Server에 저장된 홈쇼핑 미체결 고객 DB를 조회하여, 홈쇼핑 TM 팀 담당자에게 라운드로빈 방식으로 균등하게 배정한 뒤 결과를 DB에 일괄 저장하고 엑셀 파일로 출력하는 업무 자동화 도구입니다.

🔒
중앙 집중 설정 관리

DB 접속 정보를 Google Drive의 Config_DB.db로 관리하여 소스코드에 비밀번호를 노출하지 않습니다.

🔄
라운드로빈 배정

여러 담당자에게 입력한 건수만큼 순서를 돌아가며 공평하게 배정합니다.

⚡
백그라운드 처리

DB 조회 중에도 화면이 멈추지 않습니다. 조회 완료 후 처리상황 로그로 결과를 확인합니다.

📊
엑셀 자동 저장

DB 입력 완료 시 전체 시트와 담당자별 시트로 구성된 엑셀 파일을 자동 생성합니다.

실행되는 쿼리 (DB 내용)

SQLSELECT DISTINCT

    AssignDate, OrderNo, HS_Name
FROM dbo.TM_MEMBER
WHERE LEFT(Rec_Date,7) IN ('2021-10', '2022-02', ...)
ORDER BY Rec_Date, ID

SQLSELECT SaBun, SaName FROM Staff

DB 배정
담당자별 배정 건수를 입력하고 자동 배정합니다 — 화면 처리만, DB 변경 없음

실행 방법

  1. ③ 배정 테이블에서 배정할 담당자의 배정갯수 셀을 더블클릭합니다.
  2. 배정할 건수를 숫자로 입력하고 Enter를 누릅니다.
  3. 배정할 모든 담당자에 대해 1~2번을 반복합니다.
  4. DB 배정 버튼(주황색)을 클릭합니다.
📏
배정갯수 합계 초과 시

배정갯수 합계가 DB 내용 건수를 초과하면 경고 팝업이 표시됩니다. 배정 수량을 조정 후 재실행하세요.

배정 후 화면 변화

② DB 내용 테이블의 상단 행부터 순서대로 신규배정일과 신규담당자가 자동 입력되며, 배정된 행은 녹색 배경으로 강조됩니다.

DB 입력
배정 정보를 실제 DB에 저장하고 엑셀 파일을 생성합니다

실행 방법

  1. STEP 3의 배정 결과를 충분히 검토합니다.

  3. 확인 팝업에서 배정 건수를 최종 확인하고 "예"를 클릭합니다.

6 배정 방식 상세 설명

▲ 라운드로빈(Round-Robin) 배정 방식 예시

동일한 우선순위의 담당자들에게 순서를 돌아가며 배정하는 방식입니다. 예를 들어 가미숙 4건, 박미아 2건을 배정하면 아래와 같이 진행됩니다.

순번배정 담당자가미숙 잔여박미아 잔여
1번째가미숙32
2번째박미아31
3번째가미숙21
4번째박미아20
5번째가미숙1완료
6번째가미숙0완료

배정 규칙

  • DB 내용 테이블의 첫 번째 행부터 순서대로 배정됩니다.
  • DB 내용은 Rec_Date, ID 기준으로 정렬되어 있습니다.
  • 배정갯수가 소진된 담당자는 자동으로 건너뜁니다.
  • 배정 총 건수 이후의 행은 배정되지 않습니다 (신규배정일·신규담당자 공백).

10 자주 묻는 질문 FAQ

DB 배정 후 배정 내용이 틀렸습니다. 어떻게 하나요?

DB 입력 버튼을 누르기 전이라면 상세 조회 버튼을 다시 클릭합니다. DB 내용 테이블이 초기화되므로 배정 테이블의 갯수를 다시 입력하고 DB 배정 버튼을 재실행하세요.

배정갯수 합계가 DB 조회 건수보다 적어도 되나요?

가능합니다. 배정되지 않은 행은 신규배정일, 신규담당자 컬럼이 공백으로 남습니다. DB 입력 시에는 배정된 행만 UPDATE됩니다.

엑셀 파일이 저장되지 않습니다.

Downloads 폴더에 동일한 파일명이 Excel에서 열려 있는 경우 저장이 실패할 수 있습니다. Excel에서 해당 파일을 닫고 DB 입력을 다시 실행하세요.

프로그램 실행 후 "설정 로드 완료" 메시지가 나오지 않습니다.

처리상황 로그에서 빨간색 오류 메시지를 확인하세요. Google Drive 접근 문제라면 인터넷 연결을 확인하고, GDRIVE_URL의 FILE_ID가 올바른지 점검하세요.

SQL Server 2008 R2에서 어떤 ODBC 드라이버를 써야 하나요?

SQL Server Native Client 10.0 사용을 권장합니다. Config_DB.db의 DB_Type 컬럼값을 드라이버명과 정확히 일치시켜야 합니다. ODBC Driver 17도 대부분 호환되지만, 구버전 서버와의 연결 안정성은 Native Client 10.0이 더 높습니다.

동일한 날짜에 여러 번 DB 입력을 해도 되나요?

파일명이 같으므로 Downloads 폴더의 기존 엑셀 파일이 덮어씌워집니다. 필요하면 기존 파일을 먼저 다른 이름으로 저장해 두세요.

오류 발생 시 개발팀에 어떻게 문의하나요?

처리상황 로그를 캡처(화면 오른쪽 하단)하여 개발팀에 전달해 주시면 빠른 지원이 가능합니다. 오류 발생 시 빨간색으로 표시된 전체 메시지를 포함해 주세요.
